# Resend Email Setup for Fratello Hub

This handoff sets up the email account used by the Hub time-off notifications.

## What Chris Needs To Create

- A Resend account at `resend.com`
- A verified sending domain for `fratellocoffee.com`
- A Resend API key
- Three Netlify environment variables:
  - `RESEND_API_KEY`
  - `EMAIL_FROM_ADDRESS`
  - `EMAIL_FROM_NAME`

The Netlify Functions in this build do not add `firebase-admin` to `package.json`. They use the existing Firebase service account environment variables over the Firestore REST API:

- `FIREBASE_SERVICE_ACCOUNT`
- `FIREBASE_PROJECT_ID`

## Recommended Values

- `EMAIL_FROM_ADDRESS`: `hub@fratellocoffee.com`
- `EMAIL_FROM_NAME`: `Fratello Hub`

## Step-By-Step Resend Setup

1. Go to `resend.com` and create or sign in to the Fratello account.
2. In Resend, open the domain setup area and add `fratellocoffee.com`.
3. Resend will show DNS records for domain verification. These usually include DKIM records, SPF or return-path records, and sometimes a verification record.
4. Add those DNS records wherever `fratellocoffee.com` DNS is managed.
5. Return to Resend and wait for the domain to show as verified.
6. Create an API key for sending email.
7. Copy the API key.
8. In Netlify, open the Fratello Hub site.
9. Go to site settings, then environment variables.
10. Add:
    - `RESEND_API_KEY`: the API key from Resend
    - `EMAIL_FROM_ADDRESS`: `hub@fratellocoffee.com`
    - `EMAIL_FROM_NAME`: `Fratello Hub`
11. Save the environment variables.
12. Redeploy the site so the functions can read the new values.

## Testing Notes

After the variables are added and the site is redeployed, submit a test vacation request as a non-owner user. The approver should receive an email with a review link. The Hub should also write a record to the Firestore `notifications` collection.

Calendar feeds are generated by `/.netlify/functions/calendar-ics?token=...`. The brief asks for `/api/calendar.ics?token=...`; that friendly alias will need a redirect or route rule if another agent has not already added one.
