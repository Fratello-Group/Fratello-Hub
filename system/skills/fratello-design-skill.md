## PART I — BRAND FOUNDATION

---

### 1. THE CORE PRINCIPLE

Fratello is a 50-year Calgary coffee institution. Not a lifestyle brand. Not a trendy roaster. A company that has been doing one thing exceptionally well for half a century.

That's the strategic foundation. Every visual reflects that without performing it.

**The test for every design — before delivery:**
> Would a 50-year institution with nothing to prove publish this?

If it looks like a brand trying to be cool — restart.
If it could belong to any coffee company — restart.
If it performs heritage instead of embodying it — restart.
If it looks like a Canva template — restart.

---

### 2. THE THREE VISUAL ARCHETYPES

Mirror the copywriting archetypes. Both must align in any piece — the words and the visual must come from the same archetype.

#### PRIMARY: THE REGULAR GUY (90% of work)
**Visual values:** Authenticity · Belonging · Confident calm

Dark Minimal Editorial. Clean typography. Generous whitespace. Real photography. Product as hero. Subtle teal accent. No tricks, no gradients, no decoration.

**Reference outputs:** Ethiopia Shantawene email · Partner Portfolio v10 · Goodmother landing page · Partner deck for Joey's

#### SECONDARY: THE SAGE
**Visual values:** Wisdom · Specificity · Earned authority

Editorial layouts with data. Specs, altitudes, processing methods called out. Origin photography, hands working, the green coffee, the roast curve. Information dense without being busy. This is where the 30-year Probat goes, the cup-of-excellence badge, the altitude callouts.

**Reference outputs:** Ethiopia G1 single-origin page · Partner Portfolio sourcing pages · Wholesale fact sheet · Lynx Ridge B2B pitch deck

#### ACCENT: THE OUTLAW
**Visual values:** Disruption · Confidence · Anti-trend

Used sparingly. A single bold typographic statement. Negative space as weapon. The unexpected crop. Black and white. Used for limited edition, anti-Big-Coffee statements, contrast pieces.

**Reference outputs:** Outlaw single origin packaging · "Decaf reputation fixed" ad · Anniversary statement piece

### 3. BANNED VISUAL ELEMENTS

| Element | Why |
|---|---|
| Gradients (background, button, text) | Trendy. Dates fast. Not Fratello. |
| Color blobs as decoration | Canva template signal. |
| Burlap sack textures, jute backgrounds | Coffee-shop cliché. |
| Chalk fonts, "handwritten" script fonts | Generic coffee aesthetic. |
| Stock photography (Unsplash coffee, generic latte) | Use Rod's library only. |
| AI-generated humans | Never. Real photography. |
| Drop shadows on text (in static design) | Cheapens the type. (Exception: video text overlay — see §11.4.) |
| Multi-color rainbow palettes | Color discipline is the brand. |
| Emoji-driven layouts (☕✨🔥) | Generic IG noise. |
| "Latte art" close-ups (heart, rosetta cliché) | Done to death. |
| Faded vintage filter, sepia | Performance of heritage. |
| Bevels, glows, 3D text effects | Never. |
| Inner shadows or text-stroke effects | No. |
| More than 2 typefaces in one piece | Discipline = brand. |
| Centered text blocks longer than 2 lines | Hard to read. Editorial = left-aligned. |
| Decorative dividers (squiggles, dots, ornaments) | One thin line is enough. |
| Cream callouts inside dark/teal/black documents | Cream breaks the palette. Use grey card with teal accent instead. |

### 4. THE "SHOW, DON'T TELL" VISUAL RULE

Mirror of the copywriting rule, applied visually.

| Don't TELL | DO SHOW |
|---|---|
| Badge that says "Premium" | Premium photography, premium paper feel, premium spacing |
| "50 Years of Excellence" stamp | A photo of the Probat from 1962, still running |
| "Award-Winning" callout in red | A small teal eyebrow — "Cup of Excellence Top 7, 2026" |
| "Family-Owned" with a heart icon | A real photo of Chris and Russ in the roastery |
| "Hand-Roasted" with a chef hat | The roast log on Probat #2 |
| "Limited Edition" sunburst burst | Restraint. White space. A small lot number. |

The visual must do the work. If a badge or graphic icon is the only thing carrying the message, the design has failed.

#### 5.5 Color Distribution Rule

Every Fratello piece should follow approximately:

> **60% Dark or Cream (BG)** · **30% Body text** · **10% Teal accent**

Teal is **always an accent** — never a background for large surfaces, never as body text. Gold is **rare** — only for premium contexts where it's earned.

**Never:** Use both teal and gold on the same piece unless the piece is specifically about a premium product (Cup of Excellence, Goodmother, anniversary).

**The cream warning:** Inside a dominantly dark/teal/black document (status reports, B2B decks, internal proposals), do NOT introduce a cream callout — it always breaks the palette and reads as a Canva template artifact. If you need a soft callout block, use `--card-bg-dark` (#232323) with a 3px teal left border instead.

#### 6.1 The Font Stack

| Family | Use | Source | Fallback |

#### 6.2 Montserrat Weight Hierarchy

| Weight | Use |
|---|---|
| 900 (Black) | Wordmark, hero titles, large display |
| 800 (ExtraBold) | Stat numbers, page titles |
| 700 (Bold) | Section headers, eyebrows, CTAs |
| 600 (Semibold) | Subheadings, navigation |
| 500 (Medium) | Lead paragraphs, button text |
| 400 (Regular) | Body text — the workhorse |
| 300 (Light) | Hero contrast lines, captions, large display secondary |

#### 6.3 Type Scale (Documents & Presentations)

| Element | Size | Weight | Tracking |
|---|---|---|---|
| Hero title (cover) | 42–60pt | 900 | -1px |
| Page title | 28–36pt | 800 | -0.5px |
| Section header | 18–22pt | 700 | 0px |
| Eyebrow (label above title) | 7–8pt | 700 | 5–6px (UPPERCASE) |
| Stat number | 32–48pt | 800 | -1px |
| Stat label | 6–7pt | 700 | 2px (UPPERCASE) |
| Body text (document) | 9–11pt | 400 | 0px (1.6–1.7 line-height) |
| Body text (presentation) | 14–18pt | 400 | 0px (1.5 line-height) |
| Caption / fine print | 7–8pt | 400 italic | 0px |
| Footer | 7–8pt | 400 | 0px |

#### 6.4 Type Scale (Web & Social)

| Element | Web | Social Square | Story 9:16 |
|---|---|---|---|
| Hero | 64–96px | 80–120pt | 100–140pt |
| Title | 32–48px | 50–70pt | 60–80pt |
| Body | 16–18px | 28–36pt | 32–40pt |
| Eyebrow | 11–13px | 16–20pt | 18–22pt |

#### 6.5 Typography Rules

- **Tracking on UPPERCASE:** Always 4–6px letter-spacing for any all-caps text (eyebrows, labels, footer items). Lowercase Montserrat needs zero tracking.
- **Line-height:** 1.6–1.7 for body, 1.2 for headings, 1.4 for subheads.
- **Italics:** Reserve for disclaimers, captions, and Sage-archetype attribution. Never italicize headlines.
- **Bold for emphasis in body:** Use teal-colored bold (`color: var(--teal)`), not just bolder weight. Pulls the eye.
- **Hero contrast trick (use sparingly):** First line in 900, second line in 300 — same size. Creates editorial tension.
- **Cover title rule:** 52pt/800 IS the design — don't reduce it. Title breaks across 2–3 lines intentionally for visual rhythm.

#### 7.3 Logo Rules

- **🚫 THE CONTRAST RULE — non-negotiable:** NEVER place black logo on dark/black background. NEVER place white logo on light/white background. The same applies to favicons. The logo must always have visible contrast against the surface beneath it. If in doubt, use the Teal variant — it works on both light and dark.
- **Clear space:** Minimum padding around logo = height of the "F" in FRATELLO. No exceptions.
- **Minimum size:** Wordmark not smaller than 90px wide on web, 0.8in on print.
- **Never:** Stretch, rotate, recolor outside palette, add effects, layer over busy photography.
- **On photography:** Always use a 30% black gradient overlay where the logo sits, or place on a flat color block.
- **In Canva templates:** LL icon top-right, 88×93px standard size for IG 1080×1350.

#### 7.4 Logo File Reality (CRITICAL — read before any HTML/PDF logo work)

All Fratello logo PNGs are RGB without alpha channel. Used naively on dark backgrounds, they create white rectangles, black boxes, or visible baseline lines. Every logo issue we've ever shipped has been in the file itself, not in CSS.

**File verification:**
- The file at `/mnt/project/Fratello_Logo_White.png` may be **black letters on white background** (filename is misleading). Always verify visually before using.
- The reliable source is the version uploaded by Mateo confirmed as "letras blancas" (white letters on dark/black background).
- `Fratello_Logo_Black.png` (any source) = entirely black pixels = unusable directly without RGBA processing.

#### 7.5 RGBA Processing Pipelines

**For dark backgrounds (headers, dark slides):**

```python
from PIL import Image
import numpy as np, base64, io

img = Image.open('Fratello_Logo_White.png')  # white letters, dark background
rgba = img.convert('RGBA')
data = np.array(rgba)
brightness = data[:,:,:3].mean(axis=2)
data[:,:,3] = np.clip(brightness, 0, 255).astype(np.uint8)
out = Image.fromarray(data, 'RGBA')

buf = io.BytesIO()
out.save(buf, format='PNG', optimize=True)
logo_b64 = base64.b64encode(buf.getvalue()).decode()
```

Result: black bg → transparent, white letters → opaque. **White logo on any dark background.**

**For white backgrounds (covers, light sections):**

```python
img = Image.open('Fratello_Logo_White.png').convert('RGBA')
data = np.array(img, dtype=np.float32)
brightness = data[:,:,:3].mean(axis=2)

h, w = data.shape[:2]
out = np.zeros((h, w, 4), dtype=np.uint8)
out[:,:,0] = 0   # R black
out[:,:,1] = 0   # G black
out[:,:,2] = 0   # B black
out[:,:,3] = np.clip(brightness, 0, 255).astype(np.uint8)

result = Image.fromarray(out, 'RGBA')
buf = io.BytesIO()
result.save(buf, format='PNG', optimize=True)
logo_b64 = base64.b64encode(buf.getvalue()).decode()
```

Result: black bg → transparent, white letters → black. **Black logo on any white background.**

1. `font-size` / `line-height` not set to 0 on the container
2. `margin-bottom` on the img itself

### 9. COMPOSITION PRINCIPLES

These four principles apply to every Fratello visual. They are what move work from 7/10 to 9.8/10.

#### 9.1 Grid Math — Every Spacing Value Is a Multiple of 8pt
No arbitrary spacing values. Pick from `8 / 16 / 20 / 24 / 32 / 40 / 48 / 64 / 80 / 96 pt` and stick to those. Section padding = 20–32pt. Card padding = 12–20pt. Page margins = 36–48pt. Define `M` as a Python or CSS variable and reuse it on every page/slide.

#### 9.2 Negative Space As Design
Whitespace groups content (Gestalt proximity). Sections need room to breathe. Minimum 18pt between major sections. The cover page has 60pt of air after the logo before the eyebrow — that air is the design.

#### 9.3 Typographic Contrast Is The Hierarchy
KPI numbers at 26–48pt/800 weight dominate. Section tags at 5.5–8pt/700 create hierarchy. The contrast between 48pt and 5.5pt IS the design. If everything is medium-sized, nothing reads as primary.

#### 9.4 60-30-10 Color Discipline
60% dark or cream BG · 30% body text · 10% teal accent. When in doubt, **remove a teal element, don't add one**. Gold appears at 0–5% (premium-earned only) and never alongside teal except in a Cup-of-Excellence-level context.

#### 9.5 Z-Pattern (Cover Pages)
The eye travels: top-left (eyebrow + title) → top-right (logo) → bottom-left (meta) → center (teal rule). Design covers to this path.

#### 9.6 F-Pattern (Data Pages)
The eye scans left edge first, then horizontally across the top, then drops down the left edge. Section tags on the left. Numbers and badges on the right. Never reverse this.

#### 11.2 AI Image Generation Rules (Gemini Pro)

These rules are validated through the Goodmother campaign (April 2026) and apply to every Gemini image generation for Fratello:

- **One reference image per prompt.** Multiple reference images confuse Gemini and produce inconsistent output. If you need to combine elements, regenerate from scratch with one composite reference.
- **State "9:16 VERTICAL PORTRAIT" twice in the prompt.** Once early, once late. Without repetition, Gemini drifts toward landscape framing.
- **Never ask Gemini to "edit just one element."** It regenerates everything and breaks what was working. If something needs changing, regenerate the entire frame from scratch with an updated prompt.
- **Always upload real product photos as reference.** Without them, Gemini invents text on labels and the result is garbled.
- **Warmth comes from light direction, not from a filter.** If output looks too yellow, the fix in the prompt is: "warmth comes from LIGHT DIRECTION not from a color filter. Whites stay white. Darks stay dark."
- **Always download as PNG** for maximum quality as reference material for downstream work.

#### 11.3 Animation Production Rules (Kling 1.6 Pro)

- **Single-frame mode only.** Image-to-image causes objects to morph unnaturally between frames — Kling reinterprets the scene. Always animate each frame individually.
- **Motion budget:** subtle handheld camera drift + wispy steam from coffee. Nothing else moves. Bags don't move. Mug doesn't move. Only camera micro-drift and steam dissipation.
- **Animation prompt template:** *"Static scene. Only two things move: thin steam rises slowly from the coffee mug, and the camera has a very subtle handheld drift — barely perceptible. Nothing else moves. Photorealistic. 4 seconds."*
- **Join frames in CapCut with HARD CUTS,** never dissolves. Hard cuts read as real camera edits between angles — dissolves read as AI animation.

#### 11.4 Video Typography (CapCut)

| Layer | Font | Color | Use |
|---|---|---|---|
| Narrative / informational text | Montserrat Bold | `#F6F1E9` cream | Main on-screen copy |

| CTA card final | Montserrat Bold + Cormorant Italic | cream + gold | Pricing block + URL |

**Rules:**
- Drop shadow 30% opacity, blur 4px on every text overlay. **No colored backgrounds behind text** — they read as amateur.

- Music level: −14dB. Music enters at 4.5s on Reveal-style ads (let the visual carry the open).

- **Cloned audio plays over B-roll only, NEVER over the talent's face.** This eliminates lip-sync risk entirely. ### 12. DOCUMENTS (PDF / DOCX)

#### 12.0 THE DOCUMENT COLOR RULE — Black, White, Teal Accent. That's it.

**This is a non-negotiable constraint that overrides the general palette in §5.**

Documents (PDF, DOCX, reports, briefs, proposals, partner portfolios, status updates, internal documents, fillable forms) are restricted to a **3-color system**:

| Role | Hex | Usage |
|---|---|---|
| **Black** | `#1A1A1A` (or `#111111` for body text) | Headers, rules, body text, structure |
| **White** | `#FFFFFF` | Page background (white mode) · Text on dark |
| **Teal** | `#36B3AF` | Section tags, live badges, deadlines, divider rules — the ONLY accent |
| **Light grey** | `#888888` | Muted labels, secondary text — ONLY on white background |
| **Border grey** | `#E0E0E0` | Thin dividers, table borders |

**No gold. No cream callouts. No gradients. No additional accent colors.** Not in the cover, not in the body, not in the footer, not in tables or charts. If a document needs a soft callout block, use a grey card (`#F5F0E8` or `#232323` in dark mode) with a 3pt teal left border — never a colored fill.

**Why this rule exists:** Documents move through Chris and Russ, partners, sales reps, and printers. They live longer than any other deliverable, get printed and reprinted, get screenshot and forwarded. Restricting the palette to 3 colors guarantees consistency across all that movement and prevents the "Canva template" feel that creeps in the moment a designer reaches for a fourth color.

**The only exception:** Documents that are explicitly part of a campaign visual system (e.g. the Goodmother landing page brief, the Mother's Day partner pitch) MAY use the campaign's locked palette. Mateo will say so explicitly when this applies. **Default = strict 3-color.**

**Presentations are different — see §13.** Decks for B2B pitches, internal strategy, and partner meetings can use the full v3.1 palette (gold for premium contexts, cream cards on white-mode slides, photography color, etc.) unless Mateo specifies otherwise.

#### 12.2 WeasyPrint Hard Constraints

These rules exist because of 15+ iterations of trial and error. Skip none of them.

| CSS Feature | WeasyPrint | Notes |
|---|---|---|
| `display:table` | ✅ Reliable | Use for ALL multi-column layouts |
| `display:flex` | ⚠️ Unreliable | Avoid. Use tables instead |
| `display:grid` | ❌ No | Not supported |
| `position:fixed` | ✅ Works | For footers, but `@page` is better |
| `@page` margin boxes | ✅ Reliable | Best for headers/footers |
| `calc()` | ✅ Works | — |
| `page-break-inside:avoid` | ✅ Essential | Put on every card/block |
| PNG with RGBA | ✅ Works | Process logos to RGBA via §7.5 |
| Google Fonts via URL | ❌ Blocked | Network blocks all domains except npm/pypi/github. Must embed base64. |
| `print-color-adjust:exact` | ✅ Required | Without it, dark backgrounds print as white |
| `margin` on table cells | ❌ Wrong | Use `padding` on `td`, never `margin` |
| `background` on `@bottom-*` | ❌ Wrong | Causes persistent black band artifact. Footer must have white background. |

#### 13.6 PPTX Editing Pipeline

```bash
# 1. Unpack the pptx into editable XMLs
python /mnt/skills/public/pptx/scripts/office/unpack.py original.pptx unpacked/

# 2. Edit XMLs directly in unpacked/ppt/slides/slideN.xml
#    (use grep to find specific text, str_replace to swap)

# 3. Clean (removes thumbnail caches, fixes references)
python /mnt/skills/public/pptx/scripts/clean.py unpacked/

# 4. Repack — IMPORTANT: pass --original to preserve embedded media
python /mnt/skills/public/pptx/scripts/office/pack.py unpacked/ output.pptx --original original.pptx

# 5. Convert to PDF for visual verification
python /mnt/skills/public/pptx/scripts/office/soffice.py --headless --convert-to pdf output.pptx

# 6. Render slide thumbnails for QA
pdftoppm -jpeg -r 100 output.pdf slide
# Then `view slide-01.jpg`, `view slide-02.jpg`, etc.
```

**Critical EMU unit awareness:** PowerPoint uses English Metric Units. Slide width at 16:9 standard = 9144000 EMU. When editing position/size in XML, all values are in EMU (1 inch = 914400 EMU, 1 cm = 360000 EMU).

**Cowork limits for PPTX work:** never combine in a single Cowork run: pixel-level image analysis + mass file rename + Drive API + EXIF extraction. Gate sequentially with human checkpoints. Close all other Claude instances before running. Limit Sheets operations to 30 rows per batch.

- Text-to-image ratio Facebook-safe (<20%)

- Always test 3 variants minimum (paid-media skill calls for 3–4 angles — design must support all)

#### 16.4 Three Data-Validated Creative Laws

These are not opinions — they are observed in Fratello's own ad data and override aesthetic preferences.

1. **Placement > creative quality.** Same creative tested across placements: 5.30× ROAS in remarketing vs 1.69× in acquisition. Match the creative to where it runs, not to which one looks best in isolation.

2. **Brand-first > price hooks.** Same product, same audience: organic-style video 5.33× ROAS vs price-led hook 2.11×. Price-led design destroys ROAS. Lead with brand and product, not with discount.

3. **Nespresso Evergreen works cold/broad but fails with existing customers.** 11.15× in acquisition, 0× in remarketing. Conquest-themed creative is for cold audiences only — never recycle into warm/retargeting.

#### 16.5 Banned in Paid Copy

- The word "subscription" — kills ROAS by frame alone. Use identity/habit framing instead ("Never run out", "Always fresh on the shelf").
- Discount-led headlines in cold/prospecting (Layer 1) — train customers to wait.
- Price + "limited time" combo without a real deadline — reads as fake urgency.

#### 17.3 Canva MCP — Known Limitations

These are documented through repeated trial:

- **Images marked `editable: false`** in Canva templates **cannot be modified via the API.** Manual finishing in Canva editor is required.

- **`start-editing-transaction` → `perform-editing-operations` (batched) → `commit-editing-transaction`** is the only reliable mutation pattern.
- Complex Canva designs almost always require manual finishing after the API generates initial structure.

#### 20.3 PIL Texture Generator (warm noir fallback when no real photo available)

Used when a layout calls for a photographic background but no Rod asset is available — generates abstract textures consistent with the brand aesthetic.

```python
import numpy as np, base64, io
from PIL import Image, ImageFilter

def make_texture(w, h, style='dark_warm'):
    arr = np.zeros((h, w, 3), dtype=np.float32)
    np.random.seed(42)
    noise = np.random.randn(h, w, 3)

    if style == 'dark_warm':
        # Roastery feel — deep browns, warm blacks
        base = np.array([18, 12, 8], dtype=np.float32)
        gx, gy = np.meshgrid(np.linspace(0,1,w), np.linspace(0,1,h))
        spot = np.exp(-((gx-0.7)**2/0.3 + (gy-0.4)**2/0.4)) * 60
        arr = base + noise * 12 + spot[:,:,np.newaxis] * np.array([1.4,1.0,0.6])
        arr[:,:,0] += 15

    elif style == 'teal_accent':
        # Dark with teal atmospheric tone
        base = np.array([10, 20, 20], dtype=np.float32)
        gx, gy = np.meshgrid(np.linspace(0,1,w), np.linspace(0,1,h))
        spot = np.exp(-((gx-0.2)**2/0.3 + (gy-0.5)**2/0.5)) * 50
        arr = base + noise * 10 + spot[:,:,np.newaxis] * np.array([0.6,1.4,1.2])

    elif style == 'coffee_grains':
        # Multiple light pools — bean texture
        base = np.array([25, 15, 8], dtype=np.float32)
        for cx, cy, s in [(0.3,0.6,0.2),(0.7,0.3,0.25),(0.5,0.5,0.4)]:
            gx, gy = np.meshgrid(np.linspace(0,1,w), np.linspace(0,1,h))
            spot = np.exp(-((gx-cx)**2/s + (gy-cy)**2/s)) * 45
            arr += spot[:,:,np.newaxis] * np.array([1.6,1.1,0.7])
        arr = base + arr + noise * 18

    elif style == 'white_light':
        # Light, airy — almost white with subtle warm gradient
        base = np.array([242, 238, 232], dtype=np.float32)
        gx, gy = np.meshgrid(np.linspace(0,1,w), np.linspace(0,1,h))
        shadow = (1 - np.exp(-((gx-1)**2/0.4 + (gy-0)**2/0.6))) * 20
        arr = base - shadow[:,:,np.newaxis] + noise * 4

    arr = np.clip(arr, 0, 255).astype(np.uint8)
    return Image.fromarray(arr, 'RGB').filter(ImageFilter.GaussianBlur(1.5))

def to_b64(img, quality=80):
    buf = io.BytesIO()
    img.save(buf, 'JPEG', quality=quality)
    return base64.b64encode(buf.getvalue()).decode()
```

**Always overlay a gradient on top for text readability:**

}
